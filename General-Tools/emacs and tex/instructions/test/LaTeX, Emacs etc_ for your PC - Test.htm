Try to compile the file (see the itemize further down if you do not know how to use Emacs/LaTeX together): 
c:\Program Files\texmf\doc\miktex\samples\hypertex\hyperdemo.tex

A common error message is: 

'latex' is not recognized as an internal or external command, operable program or batch file.

LaTeX exited abnormally with code 1 at Mon Jun 28 11:30:30
This happens when your PATH is not set correctly, see the preparation page. It should include c:\program files\texmf\miktex\bin for miktex to work with emacs/auctex.

Ispell might break down with the message: 

Can't open c:\usr\local\lib /english.hash
This is likely because a space slipped into your definition of the environment variable DICTDIR (see the preparation page). DICTDIR must be defined as "c:\usr\local\lib" rather than "c:\usr\local\lib ". Please, doublecheck this before writing to me!

For an excellent introduction to LaTeX, you should consult the lshort guide, but it can be hard to compile...

IMPORTANT: The lshort guide is not guaranteed to compile on your system, since the needed packages are not necesarily included in the Miktex version you have downloaded... you might get a successful compilation if you put a %-sign in front of the following lines in lshort.sty: 

%\RequirePackage{type1cm}% remove size restrictions
...
%\RequirePackage{textcomp}
...
%\RequirePackage{eurosans}
...
I will see if I can find a better test, or description... Please be patient.
  • Some versions of Miktex contains c:\program files\texmf\doc\guides\lshort-english\src. If you have this directory, open the file lshort.tex in Emacs and jump to "Press C-c C-c" below. Else, proceed.
  • If it does not exist, create the directory c:\localtexmf\tex\latex
  • Unzip lshort-4.14.src.tar.gz to c:\localtexmf\tex\latex (See the download page for the CTAN location of this file)
  • Open Emacs and choose Files/Open file and get the file c:\localtexmf\tex\latex\lshort-4.14\src\lshort.tex
  • Press C-c C-c <Return> to run latex
  • Press C-c C-l to view results of the compilation
  • Press C-c C-c <Return> to run latex again
  • Press C-c C-c <Return> to run latex again, again
  • Press C-c C-c <Return> to run View
  • Press C-c C-c FILE <Return> <Return> to run dvips (Note that the command is FILE and not Dvips as one might expect)
  • Press C-c C-c Print <Return> <Return> to run GSview (Also somewhat misleading name)
  • If you want to print the document, do it from GSview.
